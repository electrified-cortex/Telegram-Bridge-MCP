## Added

### Sub-session / child-agent infrastructure

- **`session/spawn-child` + `session/revoke-child` + `child/forward`** (Phase 1): full
  parent-child session lifecycle. Child sessions are granted a `child_capability` role,
  stored `parent_sid`, and approved through the standard operator flow. The parent
  receives the child token and SID in the spawn response.

- **Spawn-child service message chain** (landed 2026-05-24): structured onboarding for
  sub-agents and parents. Three `CHILD_ONBOARDING_*` service messages on first child
  dequeue; `CHILD_FIRST_DEQUEUE_CONFIRMED` fires back to the parent (once, targeted).
  `EventContent.origin` discriminator (`bridge` / `child_forward`) distinguishes
  bridge-originated from forwarded child events. `EXIT_STATUS` detection in
  `recordOutgoing`; self-revocation auth so a child can close its own session on exit.
  Session record fields: `firstDequeueOccurred`, `exit_status`. 145 files / 3221 tests.

- **Spawn-child breadcrumb hint + `SPAWN_CHILD_SUBAGENT_HINT`**: terse "call dequeue for
  next-step instructions" hint on the spawn response; matching service message on the
  parent's next dequeue with actionable guidance for dispatching a background sub-agent.

- **`sub-session-dispatch` skill** bundled into TMCP (migrated from EC skills repo with
  three live-implementation corrections: spawn-child return fields, numeric token form,
  revoke-child `child_token` is SID integer not string).

### Activity-file improvements

- **`action(type: "activity/file/touch")`**: server-side `utime()` touch for the session's
  registered activity file — gives agents a deterministic ping-and-pong recovery check
  post-compaction without shell round-trips.

- **`activity/file/create` gains `refresh: true`**: atomic wipe-and-recreate for
  post-compaction reconnect in a single call.

- **90-second dequeue cap** when an activity file is registered; monitor breadcrumb
  injected into onboarding.

### Channel subscription (MCP resource push)

- **`resources/subscribe` / `resources/unsubscribe`** for `telegram://inbox/<token>`:
  push `notifications/resources/updated` when messages arrive, eliminating activity-file
  polling. Channel registry uses leading-edge fire + cooldown model mirroring the
  activity-file kick gate. `resources.subscribe: true` capability advertised.

- **`notifications/claude/channel`** proprietary wake mechanism: fires alongside
  `notifications/resources/updated` for clients that support the CC channel interrupt.
  Launcher SSE forward wired so stdio-bridge instances receive channel events from the
  central HTTP server.

- Onboarding messages updated to prefer channel subscription over activity-file polling.
  `ONBOARDING_ACTIVITY_FILE_HINT` suppressed when a channel subscriber is active.

### Session management

- **`session/start` refresh flag**: `refresh: true` collapses first-boot,
  reconnect-of-live, and re-establish-after-drop into a single call path (8 new AC tests).

- **`kick-lockout` replaces `kick-debounce`**: post-kick lockout gate released by
  content-returning dequeue. Eliminates cold-start latency (instant kick when unlocked)
  and burst noise (one kick per lockout window). `profile/kick-lockout` action;
  `profile/kick-debounce` still accepted with translation response + deprecation notice.

- **Silent lifecycle mode** (`silent_lifecycle` profile setting): suppresses session
  start/close announcements for high-churn pods. Backward-compatible default.

- **`MCP_BIND` env var + 5-minute pair timeout** for HTTP bind address override.

### Reminder engine

- **`last_sent` + `last_received` reminder triggers** — new reminder semantics for
  conversational "last word" detection. `last_sent` fires N seconds after this session's
  most recent outbound `send` (reset only by new send); `last_received` fires N seconds
  after the most recent qualifying inbound (reset only by qualifying inbound), with `mode`
  parameter (`all` includes inter-session DMs, `operator` narrows to operator only).
  Independent clocks; service messages, reminder fires, reactions, and ack tickets never
  reset either timer (loop-prevention). Persistent (default) or `recurring: false`
  one-off. Profile-saveable. Reconnect catch-up: fires immediately on reconnect if
  elapsed > delay (same semantics as `startup` trigger). 16 ACs, operator-directed
  (msgs 60952-60995), Curator-stamped + Overseer-REVISE resolved. Spec at
  `tasks/40-queued/last-word-reminders-2026-05-24.md`.

### Tools / events

- **`POST /event`** external event endpoint: any participant POSTs `{ kind, actor_sid?,
  details? }`; bridge logs to `data/events.ndjson`, fans out `agent_event` service
  messages, triggers kind-mapped animations. Strict kind allow-list:
  `compacting`, `compacted`, `startup`, `shutdown_warn`, `shutdown_complete`.

- **`action(type: "react")` emoji alias fallback**: unsupported reaction emojis remapped
  to semantically close supported ones (`👂->👀`, `🤚->👍`, `🧠->🤔`, `👁->👀`,
  `🦻->👀`); returns `hint: "emoji_alias_applied"` with substituted value.

- **`send(type: "choice")` two-stage UX**: on button tap, chosen button highlights for
  ~500 ms before keyboard collapses and message updates with selection suffix.

- **`response_format: "compact"`** on `dequeue`, `send`, `ask`, `choose`, `confirm`,
  `send_new_checklist`: suppresses always-inferrable fields (~445 tokens saved per
  session). `timed_out: true` always emitted in compact mode.

- **`compacting` animation preset** replaced with 4-frame dot-family animation (dot
  drifting inward). New **`recovering` preset** (4-frame dot drifting outward). `POST
  /event` governor side-effect: preset name == event kind; `compacted` kind cancels
  active animation then shows `recovering` with 60 s timeout.

## Changed

- **`tools/monitor.sh`** rewritten with OS-native file-event APIs: `inotifywait` (Linux /
  inotify-tools) -> `fswatch` (macOS / Homebrew) -> sleep-loop fallback. Output contract
  (`kick` / `heartbeat` / `timeout`) unchanged.

- **`tools/monitor.ps1`** rewritten to use `[System.IO.FileSystemWatcher]` with
  `NotifyFilter = LastWriteTime`; `WaitForChanged` replaces 1-second poll loop; sleep-loop
  fallback retained in `.sh` for non-.NET environments.

- **`send` audio default**: TTS sends are now async by default — returns
  `message_id_pending` + `status: queued` immediately; result via `dequeue`
  `send_callback`. Pass `async: false` for synchronous behavior.

- `resolveNameTag` extracted as shared export used by `outbound-proxy.ts#buildHeader` —
  eliminates duplicated `name_tag ?? color ?? name` resolution.

- `parseIntParam` extracted as file-local helper in `dequeue-endpoint.ts`.

- `file-state.ts` module doc rewritten; removed false references to non-existent
  `MAX_INTERVAL_MS` and `ACTIVITY_SUPPRESS_MS` constants.

## Fixed

### Reminder / dequeue

- **`promoteDeferred` now called on every dequeue path**: was only called in the long-poll
  while loop, silently dropping deferred reminders on immediate-batch returns and
  `max_wait: 0` instant-poll paths.

- **`hasPendingUserContent` broadened** to all operator message types in `session-queue.ts`
  (was too narrow, missing some operator message kinds on the kick-gate check).

### Monitor / activity-file

- **`awk` -> pure-bash `while-read`** in `monitor.sh`: Debian's `mawk` buffers stdin in
  4 KB blocks, silently dropping `~30-byte` "changed" events. Pure-bash `while read`
  is line-oriented and flushes immediately.

- **`monitor.sh` skips `pwsh` on Linux**: Layer 1 was unconditionally selected when
  `pwsh` was on PATH; `watch.ps1` silently fails mtime-only bumps on Linux containers.

- **`monitor.sh` exits with explicit zero** to prevent non-zero exit codes propagating
  to callers.

- **Bundled `file-watching` skill synced** to canonical (added `OSTYPE` guard).

### Channel

- **`channel.ts _send`**: corrected MCP SDK call from non-existent `sendNotification()`
  to `notification()` on the underlying `Server` instance — channel push notifications
  were silently failing.

### File state

- **`replaceActivityFile` same-path guard**: old TMCP-owned file is no longer deleted when
  the new registration reuses the same path.

- **`replaceActivityFile` stale-timer guard**: old debounce timer cancelled before async
  cleanup; timer generation check prevents stale callback firing against replacement entry.

- **Kick debounce minimum lowered to 1 s** (was 30 s) — allows fast-cycling agents to set
  sub-30 s debounce windows via `profile/kick-debounce`.

### API surface / validation

- **`GET|POST /dequeue` async handler**: added `try/catch` + `next(err)` to prevent
  unhandled promise rejections crashing Node (Express 4 does not auto-handle async routes).

- **`GET|POST /dequeue`**: removed undocumented `connection_token` pass-through that was
  silently ignored.

- **`handleNameTag`**: backtick validation added — name tags containing `` ` `` now
  rejected with `INVALID_NAME_TAG`.

- **`import_profile` Zod schema**: backtick and newline regex on `name_tag` field.

- **`applyProfile`**: `nametag_emoji` fallback — profiles saved with old field name
  correctly migrated to `name_tag` instead of silently dropped.

- **`validateFilePath`**: path-traversal check now splits on path separators before
  matching `..` — fixes false positives on paths containing `..` as substring (e.g.
  `/var/log/tmcp..lock`).

- **`ActivityFileState`**: removed phantom `absorbedCount` legacy field.

- **`appendNewline` in `file-state.ts`**: replaced `console.warn` with `dlog("tool", ...)`
  for consistency.

- **`send(type: "question", choose: [...])`**: inline keyboard now removed in the same edit
  that records the answer; previously remained tappable after resolution.
