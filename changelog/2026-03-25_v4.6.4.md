# v4.6.4 — 2026-03-25

## Fixed

- Health-check governor prompt callback hook now captures the active session SID at send time and passes it as `ownerSid` to `registerCallbackHook`, preventing the session name-tag from switching when a different session becomes active before the user presses a button.
- Timeout/skip message hooks in `confirm` and `choose` now run `editWithSkipped` in the tool's session context (`runInSessionContext(_sid, ...)`), keeping the session name-tag consistent when the hook fires.
